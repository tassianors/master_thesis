%===============================================================================
\chapter{Introdução}
\label{chapter:intro}
%===============================================================================

A teoria de identificação de sistemas é foco de estudo a diversas décadas por inúmeros engenheiros e pesquisadores em
diversas áreas. Muitas aplicações existem que se baseiam direta ou indiretamente no uso de identificação de sistemas,
muitas outras aplicações estão em processo de desenvolvimento e outras ainda em processo de incipiência. Dentro deste
contexto, o estudo da teoria de identificação de sistemas pode ser considerado amplo, neste trabalho tem-se o interesse
em apresentar alguns pontos de interesse neste assunto que são base para o desenvolvimento deste trabalho.

Podem existir diferentes subdivisões de blocos de estudo para o tema identificação de sistemas. Neste trabalho optou-se
por fazer uma divisão em sistemas lineares e não lineares de tempo discreto, entretanto esta divisão em duas
partes não é algo que possa ser estudado completamente separado. Existem muitos pontos comuns a estes dois blocos,
principalmente no que diz respeito a ideia base para a identificação de sistemas.

Básicamente identificar um sistema, ou uma parte de um sistema maior, pode ser dividido em três componentes principais:
é necessário que exista um sistema real (denotado neste trabalho por $\mathcal{S}$); uma classe de modelos (denotado
por $\mathcal{M}$) que pretende representar este sistema real; um critério que elenca dentre os infinitos modelos que
podem ser obtidos de uma classe, qual é o que melhor representa este sismtema, baseado em alguma performance. Do sistema
real coletam-se os dados que serão utilizados para proceder o processo de identificação, muitas das propriedades das
estimativas obtidas são baseadas na qualidade e quantidade que estes dados coletados conseguem trazer do sistema real.

Nesta questão de informatividade existe uma grande e vasta quantidade de pesquisa desenvolvida e em desenvolvimento.
Questões como projetos de experimentos que neste trabalho são abordados superficialmente e que para a identificação de
sistemas são uma ferramenta formidável, possuem uma aplicabilidade em diversas áreas. Em engenharia, a escolha do sinal
que será utilizado para excitar o sistema, pode entre outras caracteristicas, reduzir erros de estimativa, reduzir tempo
e custo do processo de identificação, aumentar a confiabilidade das estimativas obtidas.

Entretanto não é apenas o sinal de entrada do sistema que determinam as propriedades estatísticas dos experimentos de
identificação executados. A escolha de uma classe de modelos para identificar o sistema é de crucial importância para
que haja sucesso no processo de identificação. A escolha desta classe deve ser tomada baseado em diversas diretrizes, o
que torna sua escolha outro foco extremamente diversificado de estudo para a teoria de identificação de sistemas.

Neste ponto é que talvez faça mais sentido um discernimento entre o que é chamado de identificação de ``sistemas
lineares'' e ``não lineares''. Se o modelo escolhido para representar o sistema real for linear, exite uma grande
quantiddade de propriedades bem definidas e métodos que asseguram as propriedades estatísticas  das estimativas obtidas.
Para o caso de identificação onde o modelo escolhido é não linear, existe uma gama talvés até maior de opções de
escolha. Muitas vezes esta quantidade exorbitante de possíbilidades acaba dificultando o entendimento do que é chamado
de identificação de sistemas não lineares, além de que devido a algumas caracteristicas que sistemas não lineares
possuem, nem todas as afirmativas existentes para sistemas lineares são válidas.

Observa-se com certa frequência que uma classe de modelos é mais performática para representar certos típos de sistemas.
Isso é devido ao fato de que uma classe de modelos ou estrutura de modelos se diferenciam umas das outras devido ao uso
de bases diferentes para representar cada sistema. Desta forma sistemas que possuem uma afinidade maior certas bases são
representados com um número menor de parametros do que quando uma base qualquer é utilizada. A busca por uma classe de
modelos simples, mas que ainda assim consiga representar o sistema real é também foco de diversos estudos. A
identificação de um sistema que não possua um número desnecessário de parametros é sempre almejado, desta forma reduz-se
a possíbilidade de existência de dinâmicas que o sistema real não possui além de reduzir a complexidade dos algoritmos
utilizados na identificação.

Algoritmos para identificação são procedimentos matemáticos que em conjunto com a definição do critério de performance,
determiam o conjunto de parâmetros da classe de modelos que melhor descrevem o sistema real. Neste aspecto existe uma
grande divisão: identificação que utiliza o método dos mínimos quadrados e os que não utilizam.

O método dos mínimos quadrados é tido como o mais simples, e devido a suas propriedades conhecidas e predefinidas é
talvez o método mais utilizado. Este método se baseia na minimização de uma função custo que é relaciona o erro do
sistema obtido com os dados coletados do sistema, minimizando o erro entre estes dois sinais. Entretanto nem todos os
modelos existentes conseguem utilizar este método e para estes casos existem inúmeras propostas de alternativas. Para
modelos que são parametrzados linearmente a utilização do MMQ é a mais utilizada, para os demais, exitem diversos
algoritmos iterativos muitas vezes que, com a adição de alguma complexidade conseguem atingir seus objetivos.

Em se tratando de aplicações na área da engenharia, o controle de sistemas tem um destaque especial no uso da teoria de
identificação de sistemas para a obtenção dos parametros do controlador desejado. Muitos dos projetos de controladores
baseados em dados se utilizam desta teoria para firmar suas bases e determinar seu funcionamento.

Assim como identificação de sistemas, projeto de controladores baseados em dados é foco de estudo em várias e diversas
linhas há décadas. Neste trabalho será dado um enfoque maior ao uso de referência virtual para a obtenção dos sinais
necessários para a identificação do controlador. Referência virtual, neste contexto é largamente utilizada pelo método
VRFT que por ser um método direto, não requer que haja um conhecimento do modelo do sistema, para que o controlador
possa ser projetado. Este método se propõm a encontrar o controlador ótimo com apenas um conjunto de dados proveniente
do sistema. 

Estas caracteristicas são bastante atrativas para diversas aplicações, sobretudo àquelas onde a parada do processo para
execução de testes e experimentos é indesejada, ou por vezes, inviável. Entretanto o uso do método VRFT possui algumas
limitações, maioria deles já endereçados em diversos trabalhos. Contudo, o uso deste método para sistemas não lineares é
ainda bastante incipiente e os trabalhos existentes nesta área exigem que haja um considerável conhecimento da planta e
do sinal aplicado para que as estimativas sejam confiáveis.

O trabalho aqui apresentado tem por objetivo endereçar uma parte deste problema. O objetivo é propor uma forma de
identificação de controladores não lineares. Para isso usou-se a ideia de referência virtual utilizada pelo método VRFT
para determinar quais são os sinais necessários para identificar o controlador ideal, que leva o sistema a se comportar
em malha fechada como especificado.

De posse destes sinais obtidos de forma virtual além daqueles medidos diretamente do sistema, utiliza-se estas
informações para alimentar um algoritmo de identificação de sistemas não lineares. Neste contexto, o sistema não linear
que se quer identificar é o controlador que atuará sobre a planta.

Para limitar o escopo do trabalho desenvolvido aqui, algumas decisões sobre estrutura de modelos foram tomadas. Optou-se
pela identificação de controladores não lineares que são representados por modelos NARMAX. Estes por sua vez são
lineares nos parâmetros, o que simplifica bastante a complexidade dos algoritmos requeridos para que a estimativa possa
ser obtida, além de que grande parte das propriedades estatísticas de identificação de sistemas são válidas e podem ser
utilizadas para determinar a qualidade dos resultados obtidos.

Para demonstrar o funcionamento desta proposta de identificação de controladores não lineares, diversos exemplos serão
abordados, tanto do algoritmo implementado quanto para o conjunto do uso de referência virtual. Os exemplos abordados
tem o intuíto de apresentar propriedades das estimativas e avaliar o desempenho da proposta para atingir seus objetivos.

O trabalho tem a seguinte organização: no Capítulo \ref{chapter:system_identification} serão abordados assuntos básicos
da teoria de identificação de sistemas. Questões como a escolha da classe de modelos, do método de identificação, das
propriedades estatisticas que as estimativas obtidas possuem e uma breve introdução ao que é conhecido como projeto de
experimentos para determinar a escolha de sinais de entrada e de objetivos de identificação. No Capitulo
\ref{chapter:nlin_si_ident} é introduzido algumas caracteristicas de sistemas não lineares. São apresentados as classes
de modelos mais comumente utilizadas para descrição de sistemas. Neste capítulo também é apresentado o algoritmo de
identificação de modelos NARMAX tanto racionais como polinomiais e alguns exemplos de uso.

No Capítulo \ref{chapter:dbcd} são introduzidos alguns conceitos sobre projeto de controladores baseados em dados.
Apresentam-se brevemente alguns dos principais métodos conhecidose estende-se a discussão para apresentar o método VRFT
e com ele a ideia de referência virtual. São também apresentados neste capítulo alguns exemplos de uso deste método e
suas propriedades para situações onde o sistema é corrompido por ruído ou quando a classe de modelos não consegue
representar o controlador ideal desejado.

No Capítulo \ref{chapter:dbnarmax} são apresentadas as contribuições deste trabalho com a união de referência virtual e
o algoritmo de identificação de sistemas não lineares que podem ser descritos por classes de modelos NARMAX. São
apresentados exemplos de uso para sistemas onde a não linearidade está na dinâmica do processo e quando está acoplaca de
forma estática na entrada ou na saída do processo. São apresentados também exemplos onde a classe de modelos não
consegue representar a totalidade das dinâmicas do controlador desejado. Por fim no Capitulo \ref{chapter:conclusion}
são apresentadas conclusões gerais sobre o que foi desenvolvido neste trabalho.
